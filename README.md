# TheFrontend API Example

Статический пример проекта для работы с API TheFrontend.

Проект можно открыть без сборки: здесь только `HTML`, `CSS` и `JavaScript`.

## Что внутри

- `index.html` - рабочая панель с input-полями, кнопками и таблицей ответа.
- `showcase.html` - красивая витрина проекта, который можно построить на этих API.
- `docs.html` - список маршрутов и обязательных полей.
- `examples.html` - короткие примеры fetch-запросов.
- `css/styles.css` - дизайн проекта.
- `js/api.js` - настройки API, список endpoint-ов и функция fetch.
- `js/app.js` - логика интерфейса.
- `js/showcase.js` - пример публичного сайта на данных из API.

## Что можно создать на этих API

В проект добавлена страница `showcase.html`. Она показывает пример красивого публичного сайта:

- категории как разделы;
- посты как карточки статей;
- выбранный пост как большая статья;
- комментарии под статьей.

Если пользователь уже ввел `API Base URL` и `token` на странице `index.html`, витрина загрузит реальные данные из API. Если ключей нет, она покажет демо-данные, чтобы идея проекта была понятна сразу.

## Как запустить

1. Скачайте проект.
2. Откройте `index.html` в браузере.
3. Вставьте `API Base URL`, например:

```text
https://your-domain.com/api
```

4. Вставьте `token` из личного кабинета вашего проекта.
5. Нажмите `Сохранить`.
6. Выберите API и действие.
7. Нажмите `Отправить запрос`.

## Как опубликовать на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Загрузите все файлы проекта.
3. Откройте `Settings -> Pages`.
4. Выберите ветку `main` и папку `/root`.
5. После публикации откройте URL GitHub Pages.

## Важно про CORS

Если проект открыт на GitHub Pages, браузер делает запросы с другого домена.
Ваш API сервер должен разрешать CORS для домена GitHub Pages.

Если видите ошибку `CORS`, нужно разрешить origin в Laravel API проекте.
